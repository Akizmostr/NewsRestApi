Response codes

As the News API is a REST API, it returns appropriate HTTP status codes. Here is a list of the status codes that you can expect from the API:

Success

200 - OK

Everything worked as expected.
204 - No Content

The request has been fulfilled but it does not need to return a response body. This notably occurs when a resource has been deleted.

Client errors

400 - Bad Request

The request was unacceptable, often due to missing a required parameter.
403 - Forbidden

The request could not be fulfilled because the client is unauthenticated, or it does not have the permission to access the resource.
404 - Not Found

The requested resource doesn’t exist.
405 - Method Not Allowed

The request method is not supported by that resource.
422 - Unprocessable Entity

The request body is invalid.

Server errors

500 - Internal Server Error

The request could not be processed due to an error on the server side.

Error Messages

For client errors, additional error messages may be provided to hint at the nature of the problem.

Error properties

The moment when the error occurred

Status code

String

The message of the error

String

The request URI

Pagination and Sorting

Pagination

All top-level API resources have support for bulk fetches via "list" API methods. For instance, you can list news and comments. These list API methods share a common structure, taking at least these two parameters: page and size.

By default, the size of the page is 10, and the first page is always returned. In order to override this behaviour, you can add the page and size parameters in the query:

http://localhost:8080/news?size=20&page=2

Response body consists of 3 parts: _embedded (contains main content), links, page.

NOTE

if the page number is out of range, the response wouldn’t contain _embedded section

Links

Links section contains available links to other pages.

{
  "first" : {
    "href" : "http://localhost:8080/news?page=0&size=2&sort=date,desc"
  },
  "self" : {
    "href" : "http://localhost:8080/news?page=0&size=2&sort=date,desc"
  },
  "next" : {
    "href" : "http://localhost:8080/news?page=1&size=2&sort=date,desc"
  },
  "last" : {
    "href" : "http://localhost:8080/news?page=2&size=2&sort=date,desc"
  }
}

Page

Page section contains page information:

Path Type Description

size

Number

The size of the page

totalElements

Number

The overall number of elements returned

totalPages

Number

The number of pages needed to list all elements

number

Number

The number of the current page

 
{
  "size" : 2,
  "totalElements" : 6,
  "totalPages" : 3,
  "number" : 0
}

Sorting

By now, sorting only by date is available. All list methods return most recent resources first (descending order). If you want to change sorting order, provide "sort" query parameter and sorting direction(asc or desc) divided by a comma.

http://localhost:8080/news?sort=date,asc

Security

Authentication

Many of News' API methods require the user to be signed in. You can perform simple GET requests without authentication. Only logged-in users can post, update, delete resources. For user registration and authentication use Users API.

The News API uses JWT token for authentication. You can get JWT token after logging in (see logging user). In order to log in successfully the user must be registered (see registration).

Authorization

In order to access secured endpoint, you must provide your JWT token in Authorization header:

DELETE /news/1 HTTP/1.1
Authorization: Bearer <token>
Host: localhost:8080

There are 3 roles of users:

subscriber
journalist
admin

The user can have multiple roles.

Only admin can assign new roles (Add roles).

Subscriber role

The subscriber is allowed to post, update and delete only his own comments.

After registration every user has the subscriber role assigned by default.

Journalist role

The journalist is allowed to post, update and delete only his own news.

Admin role

Admin has access to all private endpoints, including Add roles.

List news

Returns a list of all the news. The news are returned sorted by creation date, with the most recent news appearing first.

Query parameters

This endpoint supports pagination, sorting and search. For information about pagination and sorting refer to corresponding section.

Search parameters
Parameter Description

date

The date when the comment was originally posted

title

The title of the news

author

The name of the user who posted the news
When you have title parameter in a query, you will get all the news which have your provided string in any position of the title. Consider the following example:
GET /news?title=country
This query can return you news with following titles:
"XX countryXX"
"XXcountryXX"
"XX countryside."
and so on.
Returns

Returns an object which exposes:

  • list of found news according to provided search, paging and sorting parameters

  • pagination information

  • available pagination links.

[Must not be empty], [Must have only existing roles]

false
Returns

If successful, returns string with new roles of the user.

HTTP Request Example

Consider that user with id 2 has only subscriber role.

POST /users/2/addRoles HTTP/1.1
Content-Type: application/json;charset=UTF-8
Accept: application/hal+json
Authorization: Bearer <token>
Content-Length: 43
Host: localhost:8080

{
  "roles" : [ "admin", "journalist" ]
}
HTTP Response Example
HTTP/1.1 200 OK
Content-Type: application/hal+json
Content-Length: 29
X-Content-Type-Options: nosniff
X-XSS-Protection: 1; mode=block
Cache-Control: no-cache, no-store, max-age=0, must-revalidate
Pragma: no-cache
